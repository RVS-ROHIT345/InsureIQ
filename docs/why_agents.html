Why not just one big prompt?

One LLM call guesses. Six agents decide.

The obvious approach

Paste the whole policy into one giant prompt

📄 14-page policy → 🧠 one LLM call → “analysis”
  • ✕It will invent a maturity value or a return rate — a language model predicts text, it doesn't do arithmetic.
  • ✕Four different jobs — classify, extract, calculate, judge risk — blur into one blob of reasoning.
  • ✕One bad token and the whole answer is wrong, with no way to see which part failed.
  • ✕Untestable — you can't verify a figure it made up, and every check costs an API call.
VS
InsureIQ

Six focused agents, each with one job, in a chain

📄 →
1› 2› 3› 4› 5› 6
→ 📝 report
  • ✓Every figure is computed by deterministic Python, not guessed — the model literally can't invent a number.
  • ✓Each agent has one job and one focused prompt — extraction, coverage, finance, and risk stay separate.
  • ✓Failures are isolated and inspectable — you see exactly which agent produced what.
  • ✓The whole pipeline is testable with no API key — a 192-test suite mocks the model and runs in seconds.

The difference between a demo and something you'd trust with a financial decision: numbers vs. prose — real code does the math, the model only writes the plain-English explanation around it.
